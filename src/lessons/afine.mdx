# A-FINE — Adaptive Fidelity Index with Naturalness Evaluation

<LessonMeta tags={["IMAGE QUALITY", "CLIP", "BURN PR #4894", "5–45 MIN"]} />

A-FINE is a **full-reference image quality metric**: it takes a distorted image and a reference image, runs both through a CLIP vision transformer, and produces a single number in `(0, 100)` that correlates with how a human would rate the distorted image's quality. The "adaptive" part — and the reason it was worth adding to `burn` next to LPIPS, DISTS, and FID — is that A-FINE doesn't assume the reference is itself a high-quality natural image. When the reference is unnatural (a synthetic render, a low-quality original), the metric de-emphasises faithful reconstruction and weights the distorted image's own naturalness more heavily.

<Tier level="standard">

This lesson teaches A-FINE from the prerequisites up: what image quality assessment is, where the classical and CNN-feature metrics fall short, and exactly what each of A-FINE's five components does. You shipped 1,864 lines of A-FINE into `burn` in [PR #4894](https://github.com/tracel-ai/burn/pull/4894); this is the conceptual layer that should sit underneath that code.

</Tier>

## The image-quality assessment problem

The goal of an **image quality metric** is to produce a number, given an image (or pair of images), that **correlates with what a human observer would say about the image's quality**. Sounds simple. It is not.

Two questions decide which metric family applies:

- **Do you have a clean reference image to compare against?** If yes, you're in the **full-reference (FR)** world: SSIM, PSNR, LPIPS, DISTS, A-FINE. If no, you're in **no-reference (NR)** territory: BRISQUE, NIQE, MANIQA, the standalone naturalness head of A-FINE.
- **Are you scoring a single image, or a distribution of images?** Per-image scoring is everything we've discussed so far. Distributional scoring is FID — it compares the *distribution* of features from a set of generated images against a set of real images, and is the standard for evaluating generative models like GANs and diffusion.

A-FINE lives squarely in the **full-reference, per-image** quadrant.

## The classical baselines and why they aren't enough

### PSNR — Peak Signal-to-Noise Ratio

$$
\text{PSNR}(x, y) = 10 \cdot \log_{10}\!\left(\frac{\text{MAX}^2}{\text{MSE}(x, y)}\right)
$$

PSNR is a monotone transform of pixel-space mean-squared error. It is fast, mathematically clean, and **correlates poorly with human perception**. A slightly translated sharp image gets a low PSNR even though humans see it as identical; a heavily blurred image with the same average pixel values gets a high PSNR even though humans see it as obviously degraded.

### SSIM — Structural Similarity Index

$$
\text{SSIM}(x, y) = \frac{(2\mu_x \mu_y + c_1)(2\sigma_{xy} + c_2)}{(\mu_x^2 + \mu_y^2 + c_1)(\sigma_x^2 + \sigma_y^2 + c_2)}
$$

SSIM computes structural similarity over small local patches (usually 8×8 or 11×11) and averages. It captures **luminance** (the $\mu$ terms), **contrast** ($\sigma$ terms), and **structure** (the covariance $\sigma_{xy}$) separately. Better correlation with human perception than PSNR — but still pixel-domain. A re-stylised version of the same scene has low SSIM despite being recognisable as the same scene.

## The CNN-feature family

The breakthrough idea, around 2018: **stop comparing pixels; compare features extracted by a CNN trained on ImageNet**. The argument is that humans don't compare raw pixels — we compare semantic content, and ImageNet-pretrained CNN features encode rough semantics for free.

| Metric | Backbone | What it computes |
|---|---|---|
| **LPIPS** | VGG-16 (ImageNet) | Weighted sum of squared feature differences across CNN layers; layer weights learned on a perceptual-similarity dataset. |
| **DISTS** | VGG-16 (ImageNet) | SSIM-style structure + texture ratios computed at every VGG layer, then learnedly combined. |
| **A-FINE** | **CLIP ViT-B/32** (400M image-text pairs) | The next step on the same evolutionary tree: a broader backbone, plus an explicit naturalness head, plus an adaptive fusion. |

The structural similarity Tessarix needs you to see: **A-FINE is not a different family from DISTS — it's an extension along three axes simultaneously.** Better backbone (CLIP > VGG), explicit naturalness modelling (DISTS only models fidelity), and adaptive fusion (DISTS assumes the reference is good; A-FINE doesn't).

## Enter A-FINE — the architecture

Click through each stage of the pipeline to see what it computes and why it exists.

<AFinePipeline />

The pipeline has **five computational components**, applied in order:

1. **CLIP ViT-B/32 backbone** — shared between the two images, produces a 512-d feature embedding per image.
2. **Naturalness head (NR)** — applied to each image's embedding independently, scores how natural the image looks.
3. **Fidelity head (FR)** — applied to the *pair* of embeddings, scores their similarity using an SSIM-like ratio in CLIP feature space.
4. **Two logistic calibrators** — map raw scores onto a calibrated quality scale.
5. **Adapter** — fuses the three calibrated scores into the final `(0, 100)` output, weighting fidelity against naturalness adaptively.

"512-d feature embedding" is easy to skim past. The heatmap below visualises a synthetic pair of CLIP-like embeddings — each cell is one of the 512 dimensions, colour-coded by sign and magnitude on a shared scale. Morph between them and watch both cosine similarity and the SSIM-style fidelity ratio respond — they don't track the same way, which previews why the fidelity head's structural-similarity ratio captures something cosine misses.

## The naturalness head

The NR head asks: *given this single image's CLIP embedding, how natural does this image look on its own?*

```
Input:  f ∈ ℝ^512  (CLIP feature embedding for one image)

Linear(512 → 128) → GELU → Dropout
Linear(128 → 64)  → GELU → Dropout
Linear(64 → 1)    → scalar q_nat
```

Three things worth holding onto:

1. **The same head is applied to both images.** Once to `f_d` to produce `q_nat_d`, once to `f_r` to produce `q_nat_r`. Weights shared. This is the mechanically correct read: naturalness is a property of one image; the head doesn't care which of the pair it's looking at.
2. **The activation is the erf-based `Gelu`, not QuickGELU.** The CLIP *backbone* uses QuickGELU; the naturalness head uses standard GELU. PyIQA's reference implementation is explicit about this distinction and getting it wrong is a silent error.
3. **The final-layer bias is frozen at zero in the published checkpoint.** Any constant offset in the NR head's output would be absorbed by the logistic calibrator that comes next, so the bias is not separately learnable.

<Misconception belief="There are two separate naturalness heads — one for the distorted image and one for the reference.">

There is **one** naturalness head with shared weights, applied twice. Naturalness is a property of an image alone; the head does not care which member of the pair it is looking at. Having two heads would double the parameter count for no gain — the published checkpoint ships exactly one.

</Misconception>

## The fidelity head — SSIM, but in feature space

The fidelity head asks: *given two embeddings `f_d` and `f_r`, how faithful is `f_d` to `f_r`?*

It is **not** cosine similarity, **not** L2 distance, **not** a learned MLP on `[f_d; f_r]`. It is an SSIM-style structural-similarity ratio computed across the 512 feature dimensions:

$$
q_{\text{fid}} = \frac{(2\mu_d \mu_r + c_1)(2\sigma_{dr} + c_2)}{(\mu_d^2 + \mu_r^2 + c_1)(\sigma_d^2 + \sigma_r^2 + c_2)}
$$

with $\mu_d, \mu_r$ the means and $\sigma_d, \sigma_r, \sigma_{dr}$ the standard deviations and covariance, all taken **across the feature dimensions** of `f_d` and `f_r`.

Compare to the original SSIM formula at the top of the lesson. **Same shape.** The only difference: the means and variances are taken over CLIP feature dimensions rather than image patches. The structural lineage from SSIM → DISTS → A-FINE is direct.

To build intuition for what each term in the formula actually contributes, the calculator below collapses to 8 dimensions (instead of 512). You can drag each component of the distorted vector and watch the means, variances, covariance, and assembled ratio update term-by-term:

<FidelityHeadCalculator />

<Misconception belief="The fidelity head is just cosine similarity between f_d and f_r, dressed up.">

Cosine similarity collapses to one scalar per pair using only the angle between the vectors. The fidelity head's SSIM-like ratio uses **three statistics** per vector (mean, variance, and covariance between the pair), assembled into a structured ratio that separately rewards matching luminance, matching contrast, and matching structure — directly analogous to image-domain SSIM. Two embeddings can have high cosine similarity yet differ in variance distribution; the fidelity head sees that difference.

</Misconception>

The constants `c1` and `c2` are stabilising terms preventing division-by-zero when means or variances approach zero. A-FINE sets:

$$
c_1 = c_2 = 10^{-10}
$$

**Five orders of magnitude smaller than DISTS's $10^{-6}$.** This is the single most "if you skim the paper you fail the parity test" decision in the whole metric.

> **Why.** CLIP's output embeddings are unit-norm — the final projection L2-normalises them — so the variance of a single embedding across its 512 dimensions is roughly $1/512 \approx 2 \times 10^{-3}$. A `c1` of $10^{-6}$ is small relative to that variance, but its effect on the fidelity ratio is **scale-dependent**: at the natural CLIP scale the difference between $10^{-10}$ and $10^{-6}$ is subtle but consistent, and the trained calibrator parameters in the published checkpoint were fit assuming $10^{-10}$. Substituting $10^{-6}$ shifts the ratio enough that the downstream calibrator no longer matches its training — silently. The widget below lets you sweep over feature scales and verify this empirically: at unit scale the dynamic-range loss is small; drop the feature scale to $0.2 \times$ unit and the larger `c` starts to dominate.

## The logistic calibrators

Each raw quality score (`q_nat_d`, `q_nat_r`, `q_fid`) is passed through a 5-parameter logistic curve before being handed to the adapter:

$$
s = \beta_1 \left( 0.5 - \frac{1}{1 + e^{\beta_2 (q - \beta_3)}} \right) + \beta_4 \cdot q + \beta_5
$$

This is the standard logistic-calibrator shape from the IQA literature (LPIPS uses the same form). The five $\beta_i$ are fitted on a quality-labelled dataset and shipped frozen as part of the pretrained checkpoint. Two calibrators: one shared by the naturalness scores, one for fidelity.

## The adapter — where "adaptive" lives

The adapter takes three calibrated scores and produces the final output:

- $s_{\text{nat},d}$ — calibrated naturalness of the distorted image
- $s_{\text{nat},r}$ — calibrated naturalness of the reference image
- $s_{\text{fid}}$ — calibrated fidelity between the two

It is **not** an MLP. It is a deliberately simple parametric fusion:

$$
\text{blend} = k \cdot s_{\text{nat},r} + (1 - k)
$$

$$
\text{score} = \text{blend} \cdot s_{\text{fid}} + (1 - \text{blend}) \cdot s_{\text{nat},d}
$$

where $k \in (0, 1)$ is a single learned scalar.

Read the first line carefully. **The weight on fidelity depends on how natural the reference itself is.** If $s_{\text{nat},r} = 1$ (reference is maximally natural), blend $= k + (1-k) = 1$, so the score is purely fidelity. If $s_{\text{nat},r} = 0$ (reference is fully unnatural), blend $= 1 - k$, so fidelity is down-weighted and the distorted image's own naturalness picks up the slack.

This is what makes A-FINE **asymmetric** in `(distorted, reference)`: swapping the two doesn't give the same score, because $s_{\text{nat},d}$ and $s_{\text{nat},r}$ play different roles. Classical SSIM is symmetric; LPIPS is symmetric; A-FINE is intentionally not.

The heatmap below shows the final score across the $(s_{\text{nat},r}, s_{\text{fid}})$ plane for fixed $k$ and $s_{\text{nat},d}$. **Click and drag anywhere on the heatmap** to move the cursor; sliders remain for precision. The solid cursor is your current point; the dashed cursor automatically tracks the swap-twin (same fidelity, x-coordinate at $s_{\text{nat},d}$), so both states stay visible without flipping back and forth. A closed form for the asymmetry falls out of the algebra: $\text{asymmetry} = k \cdot |s_{\text{nat},d} - s_{\text{nat},r}| \cdot (1 - s_{\text{fid}})$ — zero when $k = 0$, zero when fidelity is perfect, zero when the two naturalness scores agree.

      caption="When k = 0, blend is always 1 (always pure fidelity — no adaptation). When k = 1, blend equals s_nat_r exactly (full adaptation: a fully unnatural reference completely zeros out fidelity)."
      accent="magenta"
      onParamsChange={onParamsChange}
    />
  )}
/>

The structural reasoning: matching a synthetic-looking reference too faithfully is rewarding the wrong thing. If you're scoring an image-restoration output and the "reference" is a low-quality stock photo, A-FINE will partially fall back to "does the restoration look natural on its own?" rather than "does the restoration match the bad reference exactly?". For matched, high-quality reference pairs, the adapter is essentially a no-op and the metric behaves like a CLIP-backed structural-similarity score.

Output is sigmoid-then-multiplied-by-100 to land in $(0, 100)$ for human readability.

</Tier>

## Putting it together — one concrete walkthrough

A single (distorted, reference) pair, walked through the full pipeline. Use these numbers as a ground truth for the per-stage widgets above.

| Stage | Computation | Value |
|---|---|---|
| **1. CLIP** | f_d, f_r — 512-d unit-norm embeddings | (vectors) |
| **2. Naturalness head** — applied to f_d | Linear → GELU → Linear → GELU → Linear | q_nat,d = **0.78** |
| **2. Naturalness head** — applied to f_r (shared weights) | same head, same weights | q_nat,r = **0.42** |
| **3. Fidelity head** — SSIM-style ratio over (f_d, f_r) | (2μ_d μ_r + c₁)(2σ_dr + c₂) / [(μ_d² + μ_r² + c₁)(σ_d² + σ_r² + c₂)] | q_fid = **0.63** |
| **4. Naturalness calibrator** — applied to q_nat,d | β₁(0.5 − σ(β₂(q − β₃))) + β₄ q + β₅ | s_nat,d = **0.72** |
| **4. Naturalness calibrator** — applied to q_nat,r | same calibrator, shared weights | s_nat,r = **0.40** |
| **4. Fidelity calibrator** — applied to q_fid | second calibrator, different β's | s_fid = **0.66** |
| **5. Adapter — blend** | k · s_nat,r + (1 − k), with k = 0.55 | blend = **0.72** |
| **5. Adapter — score** | blend · s_fid + (1 − blend) · s_nat,d | score = **0.68** |
| **5. Output** | sigmoid(score · 6 − 3) · 100 — roughly | output ≈ **72** |

This is what's happening *inside* every A-FINE evaluation. The reader is now in a position to predict, ahead of running it: *given* these intermediate values, the asymmetry of this pair (score with d↔r swapped) is k · |s_nat,d − s_nat,r| · (1 − s_fid) = 0.55 · 0.32 · 0.34 ≈ 0.060 — small, because s_fid is moderate-to-high. A truly low-fidelity pair (s_fid = 0.1, same other values) would produce asymmetry ≈ 0.16 — a meaningful shift the reader can verify against the adapter heatmap above.

The whole pipeline is *deterministic given the weights*. Once the checkpoint is loaded correctly, everything downstream is a straightforward composition of pure functions. The five implementation traps below are the entire surface where that composition can silently break.

## What A-FINE doesn't do

This lesson sells what A-FINE adds; it should also be honest about what it leaves out. A-FINE is not a universal quality metric — it has a specific scope and specific failure modes outside that scope:

- **Geometric distortion.** Translation, rotation, scaling. CLIP's vision transformer is rotation-fragile and scale-fragile; a 90°-rotated version of the same image is treated as a different scene. A-FINE inherits this. For tasks involving small geometric perturbations, structural-similarity-in-pixel-space (SSIM) is more robust.
- **Saliency-weighted perception.** Humans weight different regions of an image differently — eyes, text, faces matter more than backgrounds. A-FINE has no saliency model; it weights every CLIP feature dimension equally. Metrics like CW-SSIM or specialised face-recognition metrics handle this better.
- **Temporal consistency.** A-FINE scores single images. For video quality, frame-by-frame A-FINE misses temporal artefacts (flicker, judder, motion blur inconsistency). VMAF and ITU-T J.144 family of metrics target this regime.
- **Adversarial robustness.** A targeted adversarial perturbation can change CLIP features substantially while keeping the image visually identical (or vice versa). A-FINE inherits CLIP's vulnerabilities — it is not a security-grade integrity check.
- **Out-of-distribution domains.** A-FINE was trained on natural-image distortions. Medical imaging, satellite imagery, architectural drawings, anime — the metric still produces a number, but the calibration is no longer trustworthy without re-fitting on in-domain data.
- **Distributional comparison.** A-FINE is per-image, not per-distribution. For evaluating generative models (GANs, diffusion), FID and KID are the standard — they compare the *distribution* of generated features against real-image features.

The mental model: A-FINE is a strong full-reference metric for **single-image, natural-domain, perceptual quality** assessment. Outside that envelope, reach for the appropriate specialist metric.

## Three implementation traps (the ones that catch parity tests)

This lesson has named two implementation gotchas so far; here's the consolidated list of the silent-failure traps in the burn implementation:

1. **QuickGELU vs erf-based GELU.** The CLIP backbone uses **QuickGELU** ($x \cdot \sigma(1.702 x)$), an OpenAI shortcut that approximates GELU faster on GPU. The naturalness head uses the standard erf-based `Gelu`. Substituting either for the other silently shifts every downstream activation by ~1%, which is enough to fail the parity test against PyIQA's reference scalar. The two curves are visually indistinguishable; the difference plot below makes the gap concrete.

<GeluComparison />

2. **Fused-QKV attention layout.** The CLIP attention block uses a single `Linear(d_model, 3 * d_model)` followed by `chunk(3, -1)` to split into Q, K, V. `burn_nn`'s `MultiHeadAttention` uses separate Q, K, V linears. Loading the CLIP checkpoint into the standard burn attention would require pre-splitting the weights, and the orientation of the split is non-obvious — easy to transpose the wrong direction. The custom fused-QKV attention in this PR maps the checkpoint one-to-one.

3. **`c1 = c2 = 1e-10`, not `1e-6`.** Covered above. CLIP features are unit-scale; the trained calibrator parameters assume the smaller eps. The RatioCollapseDemo widget earlier in this lesson is the diagnostic widget for this trap.
